Good Morning Puneet, A quick update if I may with regards to the Place Visit Points program. I've discovered some rather unacceptable behaviour with the program which I have invited Reece to confirm that my conclusions are correct. There are a number of behaviours which have required remediation, but the behaviour which you'll be interested in involves Google Maps API behaviour when we're using custom marker icons. (1) I've worked out why the Google Maps API marker icons are so laggy when we hover (and unhover) over them. Believe it or not... every single time you hover over a Google Maps marker which has a non Google Icon attached, the API fetches the same URL time and time again for the icon svg image. (2) Worse yet, this behaviour cannot be avoided even if we use a .png image which normally would be cached by the browser. (3) The API doesn't care. Every time we issue the marker.setIcon command the API redraws the marker's object image, and it sends out another URL hit for the svg file! (4) The Google Maps API also doesn't care if we're using 500 markers with the same icon, it shows the same appalling behaviour. For example, if we're dealing with a building that has 50 levels with an average of 25 beacons per level, the API will send 750 individual URL hits just to draw our custom icons before it shows the map image object for the first time. Setting aside the other unacceptable database backend hits which I'm working on, the "custom marker icon" problem is something you should know about.
